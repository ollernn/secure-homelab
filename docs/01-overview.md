# 01 - Project Overview

## Project Purpose

The purpose of this project is to build a secure and documented homelab environment for learning practical infrastructure, Linux administration, containerization and cybersecurity.

The first version of the project will be built as a virtual machine on my main Windows PC. This makes it possible to learn and test server administration without needing separate physical hardware from the beginning.

## What I Am Building

I am building a small Linux-based server environment that includes:

- Ubuntu Server
- Secure SSH access
- Basic firewall configuration
- Basic server hardening
- Docker and Docker Compose
- Internal services for management and monitoring
- Documentation, diagrams and screenshots

The goal is not only to install tools, but also to understand how they work together in a small server environment.

## Why This Project Matters

This project helps me develop hands-on skills that are relevant for roles within:

- IT infrastructure
- Cloud engineering
- Cybersecurity
- DevOps
- Systems administration
- Technical support and operations

By building the environment myself, I can practice real technical tasks such as configuring Linux, securing remote access, managing containers and documenting system design.

## Version 1 Scope

The first version of the project includes:

- Ubuntu Server running in a virtual machine
- SSH access from my Windows PC
- Basic Linux configuration
- UFW firewall
- fail2ban
- Docker
- Docker Compose
- Portainer
- Uptime Kuma
- Homepage
- Nginx

## Not Included in Version 1

The following parts are intentionally left for later versions:

- Dedicated physical server
- VLAN segmentation
- VPN access
- Public exposure to the internet
- Kubernetes
- Terraform
- Azure integration
- SIEM or advanced security monitoring

## Expected Outcome

At the end of version 1, I should have a working Linux server environment running in a virtual machine.

The server should be secured with basic hardening measures, run several internal Docker services and be documented clearly in this GitHub repository.

The project should also be summarized on my personal portfolio website.
